整体体验关注点

  • 首启与引导：新用户首次进入 AR 伴读/伴宠流程是否步骤清晰、无阻断性报错。
  • 存档一致性：蛋槽进度、解锁列表与成长等级是否与后台「用户详情 → 伴宠进度」一致。
  • 弱网与重进：杀进程再进、断网恢复后进度是否回滚或重复扣减。
  • 音色联动：训练完成的音色是否在故事朗读流程中可选、无 404 或空音频。

伴宠 · 蛋槽与解锁

  • 体验感：槽位百分比与 UI 提示（hint）是否同步；解锁新 mascot 后列表是否立即更新。
  • 典型示例 A：新账号 → 仅完成教程线 → 后台应可见主宠线（如 cute-dog）成长字段有更新。
  • 典型示例 B：多槽同时推进 → 刷新后台「蛋槽进度」条形图是否与端上百分比一致。
  • 异常排查：若 JSON 损坏或缺字段，后台伴宠 Tab 会显示 errors；可配合「清除伴宠数据」后让用户端重新初始化。

伴读 / 故事 · 听感与稳定性

  • 体验感：不同情感/场景下合成是否连贯、无明显截断或爆音。
  • 典型示例：同一用户连续切换 2～3 个已训练音色，检查首字延迟与队列是否堆积。
  • 与等级联动：reader_level / reader_stars 若在端上有展示，修改后台数值后需确认小程序是否拉取最新 profile（若未做实时同步，以产品约定为准）。
